This page is a single page inside a wrapper panel and page. It has two popover boxes.



Setup

  <!-- wrapper page -->
  <div data-role="page" id="type-home">
  (<!-- global header -->)
    
    <!-- fullwidth panel -->
    <div data-role="panel" data-panel="fullwidth" data-id="some_ID">
      <!-- regular JQM page -->
    </div>

    <div data-role="panel" data-panel="popover" data-id="popover1">
      <!-- pages inside popover3 -->
    </div>

    <div data-role="panel" data-panel="popover" data-id="popover2">
      <!-- pages inside popover4 -->
    </div>

  (<!-- global footer -->)
  </div>


Info

Don't forget to specify data-target="your panel ID" on all links inside the panel! Be careful not to target to data-panel, which descripes the panel type, like "popover". Since there can be more than one popover on page, the always needs to point to the data-id of the respective panel

Popover formatting is assinged by default using the class of .ui-popover. Popovers are positioned absolutely, move them around by setting CSS top/left/bottom/right and override the default format as you please.


One nested page needs data-show="first". This page will be shown first.


To keep popovers in view when scrolling long pages, just add a class of .ui-element-fixed-top or .ui-fixed-element-bottom depending on whether you want to make it behave like a fixed header or footer. If you do this, popovers will be hidden when scrolling starts and repositioned when the scroll stops. By default you have to click the popover-trigger button again to make them show up. If you want them to re-appear automatically, just add data-autoshow="true" to the panel - not the page!


By default JQM creates a single history entry for the whole page including all nested pages. You can specify a history for each panel, which supercedes the main history while the panel is visible(!) by setting data-hash="history" on the panel - not the page! Open a panel, navigate some pages, click the browser back-button - you will now go back on the panel until the first page. Do it again, but close the panel before going back. This resets the panel-history. If you open the panel again, you are back at the inital page labelled with data-show="first". Now clicking the browser back button will take you to the previously visited webpage as the panel-history.stack = 0 and therefore the JQM history takes over.
